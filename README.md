## Что это

**Windrose PSR** — это репозиторий с автономными HTML-инструментами для полевой и штабной работы в поисково-спасательных работах.

Инструменты помогают:

- **строить поисковые сетки** по карте и экспортировать их в GPX;
- **наносить рабочую разметку**: точки, линии, полигоны, радиусы, подписи, линейку;
- **сохранять и передавать проекты** через JSON;
- **анализировать ветер и перенос запаха** для задач ПСС/ПТП;
- **готовить материалы для координаторов и поисковых групп** без установки отдельного ПО.

Проект задуман как практичный набор «открыл HTML → сделал задачу → выгрузил результат».

## Быстрый старт

1. Откройте нужный HTML-файл в браузере.
2. Выберите район работы или введите координаты.
3. Настройте карту, сетку, погодные данные или режим анализа.
4. Нанесите рабочие элементы.
5. Сохраните результат в подходящем формате: **JSON**, **GPX**, **PNG**, **JPG** или **PDF**.

Ничего устанавливать не нужно.  
Достаточно современного браузера.

## 🗺 Генератор сетки

**Генератор сетки** нужен для подготовки карты района поиска.

Основные возможности:

- построение сетки по текущему экрану карты;
- ручное выделение области под сетку;
- настройка размера квадратов: 100×100, 250×250, 500×500 м или авторазмер;
- поворот сетки по истинному или магнитному северу;
- сдвиг сетки по X/Y;
- разные форматы нумерации квадратов и вершин;
- сохранение нескольких сеток на одной карте;
- выбор, редактирование и удаление сохранённых сеток;
- экспорт последней построенной сетки в **GPX**;
- загрузка GPX обратно в инструмент;
- разметка карты точками, линиями, полигонами, радиусами, подписями и линейкой;
- экспорт карты в **PNG/JPG/PDF**;
- сохранение всего проекта или выбранной области в **JSON**;
- загрузка файлов перетаскиванием на карту;
- поддержка онлайн-подложек и локальных **MBTiles**.

### Когда полезен

- нужно быстро поделить район поиска на квадраты;
- нужно передать группе только конкретный участок;
- координатор ведёт несколько районов и хочет сохранять каждый фрагмент отдельно;
- нужно подготовить GPX для навигатора или картографического ПО;
- нужно распечатать карту с сеткой и нанесением.

## 🌬 Роза ветров

**Роза ветров** помогает оценить погодные и ветровые условия для кинологической работы.

Основные возможности:

- загрузка погодных данных по выбранной точке и периоду;
- построение розы ветров;
- расчёт направления, куда могло относить запах;
- оценка стабильности шлейфа;
- разбор почасовой динамики;
- режимы задач **ПСС** и **ПТП**;
- оценка сохранности и доступности запаха;
- учёт температуры, влажности, осадков, порывов, солнца и микрометео-рисков;
- экспорт графиков, таблиц и отчётов.

### Когда полезна

- нужно понять, куда мог уходить запах в выбранный период;
- нужно быстро сравнить прошлые и прогнозные условия;
- нужно объяснить группе, почему выбран тот или иной сектор;
- нужно сохранить расчёт для отчёта или разбора задачи.

## Форматы данных

### JSON

JSON используется для сохранения рабочих данных инструмента.

В JSON могут сохраняться:

- положение карты;
- параметры подложек;
- сетки и их точная геометрия;
- точки, линии, полигоны, радиусы и подписи;
- выбранная область;
- часть проекта, если сохраняется только выделенный участок.

Такой файл можно отправить другому человеку. Он сможет открыть его в таком же инструменте и увидеть ту же рабочую разметку.

### GPX

GPX нужен для обмена с навигаторами, ГИС и картографическими приложениями.

В проекте GPX используется для:

- выгрузки поисковой сетки;
- выгрузки сетки с нанесением;
- обратной загрузки сеток и треков в инструмент.

### MBTiles

MBTiles — формат офлайн-карт.  
Он полезен, когда интернет нестабилен или район нужно подготовить заранее.

## Работа офлайн

Часть функций зависит от интернета:

- онлайн-карты;
- поиск населённых пунктов;
- погодные данные;
- внешние библиотеки, если они подключены через CDN.

Для более автономной работы можно заранее подготовить **MBTiles** и открыть HTML-файл локально. При этом браузер всё равно попросит выбрать MBTiles-файл вручную после перезапуска — это нормальное ограничение безопасности локальных файлов.

## Ограничения

Инструменты помогают подготовить и визуализировать данные, но **не заменяют решение координатора, фактическую проверку местности и локальные источники информации**.

Важно учитывать:

- погодные данные могут отличаться от фактической обстановки на месте;
- онлайн-карты могут быть неполными или устаревшими;
- экспорт зависит от доступности тайлов и CORS-ограничений источника;
- слишком большие сетки и высокая детализация экспорта могут сильно нагружать браузер;
- любые расчёты по запаху являются вспомогательной оценкой, а не гарантией обнаружения.

## Как обновлять инструменты

Обычно каждый инструмент лежит в репозитории как один HTML-файл.

Рекомендуемый порядок обновления:

1. Сделать новую версию HTML-файла локально.
2. Быстро проверить в браузере основные сценарии.
3. Заменить соответствующий файл в репозитории.
4. Проверить открытие через GitHub Pages, если он используется.
5. При заметных изменениях обновить этот README.
